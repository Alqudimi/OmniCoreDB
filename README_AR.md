تطبيق شامل متكامل لإدارة قواعد البيانات يوفر واجهة مرئية للاتصال بأنواع متعددة من قواعد البيانات وإدارتها بواجهة مستخدم حديثة وميزات متقدمة.

---

## :sparkles: نظرة عامة

**مدير قاعدة البيانات الشامل** هو أداة قوية لإدارة قواعد البيانات تتيح للمستخدمين تصفح الجداول، وعرض وتحرير البيانات، وتنفيذ استعلامات مخصصة، واستيراد/تصدير البيانات بسهولة. التطبيق يوفر تجربة قوية وسهلة الاستخدام لإدارة قواعد البيانات عبر قواعد **SQLite** و **PostgreSQL** و **MySQL**.

## :rocket: الميزات الرئيسية

### :electric_plug: إدارة الاتصالات

*   **دعم متعدد لقواعد البيانات**: يدعم **SQLite** و **PostgreSQL** و **MySQL**.
*   **الكشف التلقائي عن نوع قاعدة البيانات**: كشف ذكي من مسار الملف أو سلسلة الاتصال.
*   **اختبار الاتصالات**: التحقق من إمكانية الاتصال بقاعدة البيانات قبل الحفظ.
*   **إدارة اتصالات متعددة**: إدارة عدة اتصالات بقواعد البيانات في وقت واحد.

### :card_index_dividers: عمليات الجداول

| الميزة | الوصف |
| :--- | :--- |
| **عرض الجداول** | عرض جميع الجداول مع عدد الصفوف والأعمدة. |
| **إنشاء جداول** | تعريف جداول جديدة مع الأعمدة والأنواع والقيود. |
| **إعادة تسمية الجداول** | تغيير أسماء الجداول. |
| **حذف الجداول** | حذف الجداول مع تأكيد. |
| **إفراغ الجداول** | إزالة جميع البيانات مع الحفاظ على هيكل الجدول. |
| **تحليل الجداول** | الحصول على إحصائيات شاملة (الصفوف، الأعمدة، الفهارس، المفاتيح الخارجية). |

### :file_folder: إدارة الأعمدة

*   **عرض الأعمدة**: عرض البيانات الوصفية للعمود بما في ذلك الأنواع وإمكانية القيمة الفارغة والقيم الافتراضية.
*   **إضافة أعمدة**: إضافة أعمدة جديدة إلى الجداول الحالية.
*   **تعديل الأعمدة**: تغيير خصائص العمود (الاسم، النوع، إمكانية القيمة الفارغة، القيمة الافتراضية).
*   **حذف الأعمدة**: إزالة الأعمدة من الجداول (لـ PostgreSQL/MySQL فقط).

### :pencil: عمليات الصفوف

*   **تصفح الصفوف**: عرض بيانات الجدول مع ترقيم الصفحات (20 صفًا في كل صفحة).
*   **إدراج صفوف**: إضافة سجلات جديدة إلى الجداول.
*   **تحديث الصفوف**: تحرير السجلات الحالية.
*   **حذف الصفوف**: إزالة السجلات الفردية.
*   **بحث في الصفوف**: تصفية البيانات عبر الأعمدة النصية.
*   **فرز الصفوف**: ترتيب البيانات حسب أي عمود (تصاعديًا/تنازليًا).

### :mag: تنفيذ الاستعلامات

*   **تنفيذ استعلامات SQL مخصصة**: تشغيل أي استعلام SQL مع تلوين بناء الجملة.
*   **سجل الاستعلامات**: تتبع جميع الاستعلامات المنفذة مع الطوابع الزمنية وأوقات التنفيذ.
*   **مسح سجل الاستعلامات**: إزالة بيانات الاستعلامات التاريخية.
*   **توقيت الاستعلام**: قياس وقت التنفيذ لكل استعلام.

### :zap: العمليات المجمعة

*   **إدراج مجمع**: إدراج صفوف متعددة في عملية واحدة.
*   **تحديث مجمع**: تحديث صفوف متعددة تطابق الشروط.
*   **حذف مجمع**: حذف صفوف متعددة حسب قائمة المعرفات.

### :inbox_tray:/:outbox_tray: استيراد/تصدير البيانات

*   **تصدير إلى JSON**: تنزيل بيانات الجدول كملف JSON.
*   **تصدير إلى CSV**: تنزيل بيانات الجدول كملف CSV.
*   **تصدير تفريغ SQL**: إنشاء عبارات SQL لقاعدة البيانات بأكملها أو لجداول محددة.
*   **استيراد من JSON**: تحميل البيانات من ملفات JSON.
*   **استيراد من CSV**: تحميل البيانات من ملفات CSV مع الإبلاغ عن الأخطاء.

### :wrench: الميزات المتقدمة

*   **فحص المخطط**: تصور هيكل الجدول التفصيلي.
*   **علاقات الجداول**: عرض علاقات المفاتيح الخارجية.
*   **إدارة الفهارس**: إنشاء وإدارة فهارس قاعدة البيانات.
*   **إدارة القيود**: التعامل مع المفاتيح الأساسية والمفاتيح الخارجية والقيود الفريدة وقيود `CHECK`.
*   **الاستعلامات المحفوظة**: حفظ الاستعلامات المستخدمة بشكل متكرر مع المعلمات والعلامات.
*   **تحليل أداء الاستعلام**: شرح وتحليل الاستعلامات مع تحليل أداء مفصل.
*   **النسخ الاحتياطي والاستعادة**: إنشاء واستعادة نسخ احتياطية لقاعدة البيانات بتنسيق SQL أو JSON.
*   **مراقبة الأداء**: تتبع الاستعلامات البطيئة ومقاييس الأداء في الوقت الفعلي.
*   **التحقق من صحة البيانات**: تعريف وتنفيذ قواعد التحقق لسلامة البيانات.

### :art: واجهة المستخدم

*   **وضع مظلم/فاتح**: التبديل بين الوضعين مع الحفظ في `localStorage`.
*   **7 لوحات ألوان للمظهر**: تخصيص تجربتك.
*   **تصميم متجاوب**: يعمل على جميع الأجهزة.
*   **رسوم متحركة حديثة**: انتقالات سلسة وتأثيرات.
*   **تأثيرات الزجاج المشكل (Glassmorphism)**: تصميم واجهة جميل.
*   **مؤثرات صوتية**: ملاحظات صوتية تفاعلية.
*   **لوحات قابلة لتغيير الحجم**: مناطق جانبية ومحتوى قابلة للتعديل.
*   **مربعات حوار التأكيد**: فحوصات أمان للعمليات المدمرة.
*   **إشعارات منبثقة**: ملاحظات في الوقت الفعلي لجميع العمليات.

## :gear: التثبيت والتشغيل

### المتطلبات الأساسية

تأكد من تثبيت ما يلي:

*   **Node.js** 20 أو أحدث
*   **Python** 3.11 أو أحدث
*   **مدير الحزم uv** (للبايثون)

### البدء السريع

1.  **استنساخ المستودع:**
    ```bash
    git clone https://github.com/Alqudimi/OmniCoreDB.git
    cd OmniCoreDB
    ```

2.  **تثبيت تبعيات Node.js:**
    ```bash
    npm install
    ```

3.  **تثبيت تبعيات Python:**
    ```bash
    uv sync
    ```

4.  **تشغيل خادم التطوير:**
    ```bash
    npm run dev
    ```
    هذا سيبدأ:
    *   واجهة API الخلفية على `http://localhost:8000`
    *   الواجهة الأمامية على `http://localhost:5000`

### بناء الإصدار النهائي

1.  **بناء الواجهة الأمامية:**
    ```bash
    npm run build
    ```

2.  **بدء خادم الإنتاج:**
    ```bash
    npm run start
    ```
    سيتم تقديم التطبيق على المنفذ `5000`.

## :shield: الأمان

### تدابير الأمان الحالية

*   تمكين CORS لمرونة التطوير.
*   منع حقن SQL عبر الاستعلامات ذات المعلمات.
*   تخزين بيانات الاعتماد في الذاكرة (بدون استمرارية).
*   عزل الاتصال لكل جلسة مستخدم.

### تحسينات الأمان المخطط لها

*   نظام مصادقة المستخدم.
*   التحكم في الوصول المستند إلى الأدوار.
*   تخزين الأسرار المشفرة.
*   سجل تدقيق لجميع العمليات.
*   إدارة الجلسات وتحديد معدل طلبات الواجهة البرمجية.
